MoScoW Requirements

Following our initial research to determine our requirements, our project and therefore our requirements changed drastically. Due to problems with existing legacy code on our client's side, our client changed the requirements for the software and hardware. Instead of using the ARCC sensors, our client decided it would be more appropriate to use a pre-built fitbit and use the fitbit API to retrieve data. This posed problems, as it is not possible to deduce some statistics from the data retrieved from the fitbit API. Also, the fitbit API is intended for a web application rather a mobile application. However, as previously noted from our surveys, our application should also monitor the playersâ€™ general health and therefore the data retrieved is still relevant for our clientele.
