Gathering Requirements: First Meeting with our Client

Our first step for gathering requirements was an initial meeting with our client, Catherine Holloway. This meeting told us what software and hardware the client is intending we use at this point and what the client is expecting regarding the features of the application. We also gathered information on what other data we would need to collected in order to complete the requirements.

Information Gathered Analysis of Information
Hardware

The sensor: At this stage of the project, it was intended we use a bespoke sensor, developed within our client’s department specifically for this project.

  • bluetooth smart system on chip
  • air pressure/ temp
  • real time clock
  • micro SD card storage
  • battery level monitor
  • non-volatile flash memory
  • 9-axis intertial/ magnetometer sensor
  • Li-Ion or Li-Po battery 3.7v

    • The wheelchair: the wheelchair available for testing does not have angled wheels, but the wheelchair that will be used in the final stages will be specially for wheelchair basketball and will have angled wheels.

    • We will have the task of analysing gyroscope and accelerometer data

    • The sensor will need to be attached to the wheelchair or the player

    • Any sensor values will need to be offset in order to take into consideration the angle of the wheels.
    Software

    In order to use the sensors our client initially proposed we use, we needed to use a pre-developed application to retrieve the data.

    		 We will need to incorporate the pre-developed application to ours in order to retrieve and use the data
    Target Audience

    The application is intended for wheelchair basketball players.

    		 At this point, we only knew the application would be used for wheelchair basketball users. From this, our next step is to speak to this client-base directly to refine more specific requirements about our end user
    Features

    The application should have a user interface that displays key statistics of a wheelchair basketball game.

    		 Our client expects us to produce an application with a user interface that displays statistics. Our next step is to find the particular characteristics of the interface that would ensure it would suite our target audience.
