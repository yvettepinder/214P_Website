Gathering Requirements: Meeting with End User

At this stage, we knew what software and hardware the project required but needed to gather more information to deduce the requirements for the end product. To do this, we arranged a meeting with Pete Donnelly, a wheelchair basketball player who is part of a team that trains in Brixton. We spoke with Pete and witnessed a training session to discover the following information.

Information Gathered about End User

None of the users of the application will be elite athletes, so the the data doesn't have to be hugely precise. The users will be more concerned about the general improvement of their skills and health.

The current wheelchair basketball players are mostly aged between 15-20. From this we can assume they will all be computer literate and familiar with using a smartphone. Additionally, players will not have age-related illnesses such as joint problems so there is no risk in terms of over-exertion in this sense.

At present, all the wheelchair basketball players are male. However, it is possible for future members to be female so the application should be gender neutral.

Desired Features

MUST:

  • connect to a phone to get stats/ player profiles
  • collect the following data: distance travelled over four separate quarters, average speed, top speed
  • have an easy to use interface
  • have an option to pause data collection (for time-outs etc)

    • COULD:

  • provide heart rate data
  • monitor passes and shots
  • provide calories burnt
